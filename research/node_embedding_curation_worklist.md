# Node-embedding `conforms_to` curation worklist

Generated by the `dismech.node_embeddings` library (see
`src/dismech/node_embeddings/`). For each pathophysiology node **without** a
`conforms_to` link, we embed its text (node label + ontology term labels) and
find its nearest neighbour among the 273 **labelled** nodes; if cosine >=
threshold we propose that neighbour's module target as a candidate
`conforms_to`.

## Artifact

`research/conforms_to_suggestions.tsv` — 1,128 suggestions at cosine >= 0.70,
columns: `cosine`, `suggested_conforms_to`, `node`, `nearest_labelled_node`.

These are **leads for a curator**, not auto-applied edits. High-cosine rows are
typically near-identical node names across related disorders (e.g. cerebellar
ataxia subtypes -> `cerebellar_purkinje_degeneration#Purkinje Neuron
Degeneration`); mid-range rows are the more interesting cross-disease
generalisations. A human should confirm each before adding `conforms_to` and the
module's expected biological processes / causal edges.

## Regenerate

```bash
# default local backend (model2vec, no torch)
uv run --with model2vec --with numpy \
  dismech-node-embed suggest --threshold 0.70 --out research/conforms_to_suggestions.tsv

# or plug in OpenAI (requires OPENAI_API_KEY)
dismech-node-embed suggest --backend openai:text-embedding-3-small --threshold 0.70 --out ...
```

## Evaluation against ground truth (`conforms_to`)

`dismech-node-embed evaluate` scores embeddings on the 273 labelled nodes by
retrieving same-module neighbours.

| backend | cosine same / diff | separation | precision@1 |
|---|---|---|---|
| model2vec potion-base-8M (default) | 0.735 / 0.254 | 2.89x | **0.845** |
| model2vec potion-base-32M | 0.739 / 0.252 | 2.94x | 0.831 |

The larger static model is a wash, so 8M is the default. OpenAI
(`--backend openai:...`) is a one-flag swap and the main comparison worth
running when a key is available; it is expected to help most on the mid-range
cross-disease rows where lexical overlap is low.

## Caveats / next steps

- Suggestions inherit the curation bias of the 273 labelled nodes; modules with
  few labelled members generalise less well.
- Precision@5/@10 are capped by small positive-class sizes (~2.6 nodes per
  module target), so precision@1 is the honest headline metric.
- A natural follow-up is a confidence-banded review queue (e.g. split 0.70-0.85
  vs >0.85) and feeding accepted rows back as new labels to bootstrap coverage.
